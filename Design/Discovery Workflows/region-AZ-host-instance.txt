Design/Discovery Workflows/region-AZ-host-instance.txt
============================================
Objective
---------
fetch instance data:
- id
- name
- state
- project (tenant)
- neutron agents
- vNICs
  - id?
  - TP name

How to fetch:
--------------- 
API: 
- nova list
 -- get 'name' , 'Status' , 'Power State' , 'Networks' for each instance !
- nova show NAME   // for each instance above
 -- get hostid, id, tenant_id, 'security-groups'
 
- nova hypervisor-servers 2  // get which instance (Linux names) on which host (from hypervisor with ID=2)
 -- get 'Name' to 'Hypervisor Hostname' mappings

CLI/SSH on the controller : nova-manage vm list | column -t   to get 'real' OS instances names
CLI/SSH to each of the host_ip 's and then on each:
virsh list  // instance list and ids (Linux names)
virsh dumpxml 2 // many low level details for that instance
